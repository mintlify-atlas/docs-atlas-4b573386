## Overview

Echo employs multiple layers of content moderation to keep the platform safe and appropriate for college communities. This includes AI-powered NSFW detection, user reporting, and admin moderation tools.

## NSFW detection

Echo uses TensorFlow.js with NSFWJS to detect inappropriate images before upload.

### How it works

<Steps>
  <Step title="Client-side scanning">
    Images are scanned in the browser before upload:

    ```typescript lib/nsfw.ts
    import * as tf from "@tensorflow/tfjs";
    import * as nsfwjs from "nsfwjs";

    class NSFWFilter {
      model: nsfwjs.NSFWJS | null = null;

      constructor() {
        this.getModel();
      }

      async getModel() {
        try {
          this.model = await nsfwjs.load();
        } catch (error) {
          console.error('Failed to load NSFW model:', error);
        }
      }

      async isSafe(file: File): Promise<boolean> {
        const predictions = await this.predictImg(file, 3);
        
        const pornPrediction = predictions.find(
          ({ className }) => className === "Porn"
        );
        const hentaiPrediction = predictions.find(
          ({ className }) => className === "Hentai"
        );

        if (!pornPrediction || !hentaiPrediction) {
          return true;
        }

        // Reject if probability > 25%
        return !(
          pornPrediction.probability > 0.25 ||
          hentaiPrediction.probability > 0.25
        );
      }
    }

    export default new NSFWFilter();
    ```
  </Step>

  <Step title="Prediction categories">
    NSFWJS classifies images into 5 categories:

    | Category | Description | Action |
    |----------|-------------|--------|
    | **Drawing** | Drawings, cartoons | Allow |
    | **Hentai** | Anime/manga NSFW | Block if >25% |
    | **Neutral** | Safe content | Allow |
    | **Porn** | Explicit content | Block if >25% |
    | **Sexy** | Suggestive content | Allow |

    <Info>
      The 25% threshold balances safety with false positives. Adjust based on your community standards.
    </Info>
  </Step>

  <Step title="Usage in upload flow">
    ```typescript components/PostComposer.tsx
    import nsfwFilter from '@/lib/nsfw';

    const handleImageSelect = async (file: File) => {
      try {
        // Show loading state
        setScanning(true);

        // Check if image is safe
        const isSafe = await nsfwFilter.isSafe(file);

        if (!isSafe) {
          toast.error(
            'This image contains inappropriate content and cannot be uploaded.'
          );
          return;
        }

        // Proceed with upload
        await uploadImage(file);
      } catch (error) {
        toast.error('Failed to scan image. Please try again.');
      } finally {
        setScanning(false);
      }
    };
    ```
  </Step>
</Steps>

### NSFW detection accuracy

<Tabs>
  <Tab title="Strengths">
    - **Fast**: Runs client-side, no server round-trip
    - **Private**: Images never leave user's browser during scanning
    - **Accurate**: ~90%+ accuracy on explicit content
    - **Free**: Open-source model, no API costs
    - **Offline**: Works without internet (after model loads)
  </Tab>

  <Tab title="Limitations">
    - **False positives**: May flag artistic/medical images
    - **Context-blind**: Doesn't understand context
    - **Client-side**: Can be bypassed by determined users
    - **Model size**: ~4MB download on first use
    - **Performance**: Can slow down older devices

    <Warning>
      NSFW detection is a deterrent, not a guarantee. Combine with user reporting and admin review.
    </Warning>
  </Tab>

  <Tab title="Tuning">
    Adjust the probability threshold:

    ```typescript
    // More strict (30% threshold)
    return !(
      pornPrediction.probability > 0.30 ||
      hentaiPrediction.probability > 0.30
    );

    // More lenient (20% threshold)
    return !(
      pornPrediction.probability > 0.20 ||
      hentaiPrediction.probability > 0.20
    );

    // Include "Sexy" category
    const sexyPrediction = predictions.find(
      ({ className }) => className === "Sexy"
    );
    return !(
      pornPrediction.probability > 0.25 ||
      hentaiPrediction.probability > 0.25 ||
      sexyPrediction.probability > 0.50
    );
    ```
  </Tab>
</Tabs>

## User reporting

Users can report inappropriate posts for admin review.

### Report flow

<Steps>
  <Step title="User reports post">
    ```typescript
    // POST /api/profile/report
    const reportPost = async (postId: string, reason: string) => {
      const response = await fetch('/api/profile/report', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          postId,
          reason
        })
      });

      if (response.ok) {
        toast.success('Report submitted. Thank you for keeping Echo safe.');
      }
    };
    ```
  </Step>

  <Step title="Report stored in database">
    ```typescript app/api/profile/report/route.ts
    export async function POST(req: NextRequest) {
      const session = await getServerSession(config);
      if (!session) {
        return NextResponse.json(
          { error: 'Authentication required' },
          { status: 401 }
        );
      }

      const { postId, reason } = await req.json();

      const post = await db.post.findUnique({
        where: { id: postId },
        select: { authorId: true },
      });

      if (!post) {
        return NextResponse.json(
          { error: 'Post not found' },
          { status: 404 }
        );
      }

      // Create report
      const report = await db.report.create({
        data: {
          reason,
          postId,
          userId: post.authorId,
          reportedById: session.user.id,
        },
      });

      return NextResponse.json(report);
    }
    ```
  </Step>

  <Step title="Admins review reports">
    Admins can view all reports and take action:

    ```typescript
    // GET /api/admin/reports
    const reports = await db.report.findMany({
      include: {
        post: {
          include: {
            author: {
              select: { username: true },
            },
          },
        },
        reportedBy: {
          select: { username: true },
        },
      },
      orderBy: { createdAt: 'desc' },
    });
    ```
  </Step>
</Steps>

```typescript Report Dialog
const ReportDialog = ({ postId }: { postId: string }) => {
```

    ```typescript
    export async function DELETE(
      req: Request,
      { params }: { params: { id: string } }
    ) {
      const session = await getServerSession(config);

      if (!session?.user?.isAdmin) {
        return NextResponse.json(
          { error: 'Admin access required' },
          { status: 403 }
        );
      }

      await db.post.delete({
        where: { id: params.id },
    ```

## Automated moderation

Implement automated rules to catch common violations:

<Accordion title="Spam detection">
  ```typescript
  // Detect rapid posting
  const recentPosts = await db.post.count({
    where: {
      authorId: userId,
      createdAt: {
        gte: new Date(Date.now() - 60 * 1000), // Last minute
      },
    },
  });

  if (recentPosts > 10) {
    // Flag as potential spam
    await db.report.create({
      data: {
        reason: 'Automated: Rapid posting detected',
        userId,
        reportedById: 'system',
      },
    });
  }
  ```
</Accordion>

<Accordion title="Duplicate content">

  ```typescript
  // Check for duplicate posts
  const duplicate = await db.post.findFirst({
  ```

## Moderation workflow

<Steps>
  <Step title="Automatic screening">
    - NSFW detection blocks inappropriate images
    - Rate limiting prevents spam
    - Duplicate detection catches reposts
  </Step>

  <Step title="User reports">
    - Users flag problematic content
    - Reports queue for admin review
    - Multiple reports prioritize review
  </Step>

  <Step title="Admin review">
    - Admins view report queue
    - Review content and context
    - Decide on action:
      - Dismiss (false report)
      - Delete post
      - Warn user
      - Ban user
  </Step>

  <Step title="Action taken">
    - Post deleted if violating
    - User notified of violation
    - Repeated violations = ban
    - Appeals process (if implemented)
  </Step>
</Steps>
