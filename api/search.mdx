## Search

`GET /api/search`

Search across users, posts, and colleges using a query string.

### Query parameters

<ParamField query="q" type="string" required>
  Search query string
</ParamField>

<ParamField query="type" type="string" optional>
  Filter results by type. Options: `all`, `users`, `posts`, `colleges`. Defaults to `all`
</ParamField>

## Search behavior

### User search

Searches across:
- Usernames (case-insensitive partial match)
- Biographies (case-insensitive partial match)

**Filters**:
- Only returns users with `PUBLIC` privacy setting
- Maximum 10 results

### Post search

Searches across:
- Post text content (case-insensitive partial match)

**Filters**:
- Only returns posts with `ANYONE` privacy
- Results sorted by creation date (descending)
- Maximum 20 results

### College search

Searches across:
- College names (case-insensitive partial match)

**Filters**:
- Maximum 10 results
- Includes user count for each college

## Example requests

### Search all content

```bash
curl -X GET 'https://your-domain.com/api/search?q=computer%20science' \
  -H 'Content-Type: application/json'
```

### Search only users

```bash
curl -X GET 'https://your-domain.com/api/search?q=john&type=users' \
  -H 'Content-Type: application/json'
```

### Search only posts

```bash
curl -X GET 'https://your-domain.com/api/search?q=echo&type=posts' \
  -H 'Content-Type: application/json'
```

### Search only colleges

```bash
curl -X GET 'https://your-domain.com/api/search?q=stanford&type=colleges' \
  -H 'Content-Type: application/json'
```

## Use cases

### Autocomplete search

Implement real-time search suggestions:

```javascript
const searchUsers = async (query) => {
  const response = await fetch(
    `/api/search?q=${encodeURIComponent(query)}&type=users`
  );
  const { users } = await response.json();
  return users;
};
```

### Content discovery

Find relevant posts based on keywords:

```javascript
const searchPosts = async (keyword) => {
  const response = await fetch(
    `/api/search?q=${encodeURIComponent(keyword)}&type=posts`
  );
  const { posts } = await response.json();
  return posts;
};
```

### College exploration

Discover colleges and their user counts:

```javascript
const searchColleges = async (name) => {
  const response = await fetch(
    `/api/search?q=${encodeURIComponent(name)}&type=colleges`
  );
  const { colleges } = await response.json();
  return colleges;
};
```

## Performance considerations

- Search queries use database indexes for optimal performance
- Results are limited to prevent excessive data transfer
- Case-insensitive matching uses PostgreSQL's `insensitive` mode
- Privacy filters are applied at the database level

## Tips

- Use specific search terms for better results
- Filter by type when you know what you're looking for
- URL-encode query parameters to handle special characters
- Implement client-side debouncing for real-time search
